# Глобальное динамическое освещение картографических 3D данных

Данный репозиторий содержит реализацию алгоритма глобального динамического освещения, адаптированного для картографических 3D-данных и работающего в среде WebGPU. Проект выполнен в рамках выпускной квалификационной работы.

Основная цель проекта — интегрировать технологию Глобального Освещения (Global Illumination) в картографический движок. В отличие от стандартного (прямого) освещения, глобальное учитывает не только свет, пришедший от источника напрямую, но и вторичные лучи — свет, отраженный от окружающих поверхностей. Это позволяет добавить карте объем.

В отличие от традиционных методов (ray tracing, path tracing), требующих значительных вычислительных мощностей, предлагаемый подход позволяет получать физически корректное освещение в реальном времени на мобильных устройствах и в браузере.

**Ключевые особенности:**
- Irradiance-пробы со сферическими гармониками (9 коэффициентов)
- Вокселизация через карту высот для быстрой проверки видимости
- Динамический небосвод по модели All Sky Model
- Адаптивная и регулярная расстановка проб
- Конфигурируемые параметры освещения в реальном времени

## Как это работает

### 1. Небесная полусфера

Геометрия небосвода построена на основе икосаэдра. Яркость каждой точки рассчитывается по модели All Sky Model в зависимости от положения солнца и коэффициента ясности (`Nevg`). Полученное распределение яркости преобразуется в irradiance cubemap и используется как ambient-добавка к основному освещению.

### 2. Irradiance-пробы

Каждая проба представляет собой набор из 9 сферических гармоник (аналог преобразования Фурье для функций на сфере), описывающих входящее освещение в точке пространства. Пробы создаются с помощью кубической камеры, захватывающей окружение, после чего вычисляются коэффициенты гармоник.

### 3. Расстановка проб

Поддерживаются два подхода:
- **Регулярная сетка** — быстрее, но часть проб может попадать внутрь зданий
- **Адаптивная сетка** — пробы не попадают внутрь геометрии за счёт использования карты высот

### 4. Проверка видимости

Для каждого текселя карты высот проверяется, какие из соседних проб в нём видны. Используется растеризация отрезка (DDA-алгоритм) — если луч между точкой и пробой не пересекает геометрию, проба считается видимой.

### 5. Расчёт освещения

Финальное освещение точки вычисляется как взвешенная сумма вкладов от ближайших видимых проб с учётом:
- расстояния до пробы
- угла падения света
- коэффициента видимости

## Установка и запуск

```bash
# Клонирование репозитория
git clone https://github.com/EkatherinaS/itmo-global-dynamic-illumination-2026

# Установка зависимостей
npm install

# Запуск в режиме разработки
npm run dev

# Сборка для production
npm run build
```

## Конфигурируемые параметры

| Параметр | Описание |
|----------|----------|
| **Положение солнца** | Координаты (x, y, z) — меняет направление света |
| **Коэффициент Nevg** | Ясность неба (1.0 — чистое небо, 0.2 — пасмурно) |
| **Цвет небосвода** | Общий оттенок рассеянного света |
| **Размер сетки проб** | Количество проб по горизонтали (например, 5×5, 10×10, 20×20) |
| **Способ расстановки** | Регулярная или адаптивная сетка |
| **Интенсивность компонент** | Пробы / irradiance cubemap / прямой солнечный свет |
| **Учёт угла падения** | Включение/выключение косинусного множителя |
